The purpose of activation functions is to introduce non-linearities in the network.

  • 1. Sigmoid: The Sigmoid function squashes any real-valued number into a range between 0 and 1.
    • When to use it:
      • Output Layer of Binary Classifiers: It is the standard choice for the final layer when you need to predict a probability (e.g., "Is this email spam or not?").
    • Why it fell out of favor for hidden layers:
      • Vanishing Gradient Problem: For very high or very low input values, the gradient becomes almost zero. This "kills" learning in deep networks.
      • Not Zero-Centered: The outputs are always positive, which can make the optimization (gradient descent) zig-zag and slow down convergence.
  • 2. Tanh (Hyperbolic Tangent): Tanh is similar to Sigmoid but squashes values into a range between -1 and 1.
    • When to use it:
      • Hidden Layers (if data is centered): It is generally preferred over Sigmoid for hidden layers because it is zero-centered. This means the average of the outputs is closer to zero, which helps the next layer learn more efficiently.
      • Recurrent Neural Networks (RNNs): It is frequently used in RNNs and LSTMs to regulate the flow of information.
    • The Catch: Like Sigmoid, it still suffers from the vanishing gradient problem when the inputs are very large or small.
  • 3. ReLU (Rectified Linear Unit): ReLU is the most widely used activation function in deep learning today. It outputs the input directly if it is positive; otherwise, it outputs zero ($f(x) = \max(0, x)$).
    • When to use it:
      • Hidden Layers (Default Choice): You should almost always start with ReLU for your hidden layers. It is the standard for Convolutional Neural Networks (CNNs) and deep feedforward networks.
    • Strengths:
      • Computationally Efficient: It only requires a simple thresholding at zero (no expensive exponents).
      • Reduces Vanishing Gradients: It does not saturate in the positive direction, allowing deep models to converge much faster than Sigmoid or Tanh.
    • The Weakness:
      • "Dying ReLU" Problem: If a neuron's input is always negative, it will always output zero and its gradient will be zero. That neuron becomes "dead" and never updates again. (This is often solved using Leaky ReLU).
